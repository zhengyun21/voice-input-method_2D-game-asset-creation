# 修复翻译提示词 — 数学表达式被错误处理

## 问题描述

当用户翻译包含数学表达式（如 `6=2+2+2`）的文本时，模型输出异常——可能保留了原文或将数学表达式错误翻译。根因在于 system prompt 中的"不要输出原文"指令与数学表达式的处理产生了冲突。

## 根因分析

当前 `translateAuto` 的 system prompt（L156）：

```
你是一个专业翻译助手。请直接将用户输入的文本从{sourceLang}翻译成{targetLang}，仅返回翻译结果，不要添加任何额外解释或说明，不要输出原文。
```

**"不要输出原文"** 这条指令的初衷是：防止模型在长文档翻译时先回显原文再翻译，导致截断后用户只看到原文。

**但这条指令对数学表达式造成了问题**：

- `6=2+2+2` 这类表达式既是"原文"的一部分，也是语言中立的客观内容
- 模型可能在"不要输出原文"和"正确翻译数学表达式"之间产生歧义
- 如果用户输入："这里以6=2+2+2为例子"，模型可能把 `6=2+2+2` 当作需要处理的"原文"来对待

## 修复方案

将 "不要输出原文" 改为更精确的表述：**"不要添加原文内容，不要添加解释性文字，只输出翻译结果"**。

这样数学表达式作为原文内容的一部分会被保留（因为它们不属于"解释性文字"），同时模型仍然不会添加多余的解释。

## 实现步骤

### 修改 `deepseek.ts` — 更新 `translateAuto` 的 system prompt

将第 156 行的 prompt 从：
```
你是一个专业翻译助手。请直接将用户输入的文本从{sourceLang}翻译成{targetLang}，仅返回翻译结果，不要添加任何额外解释或说明，不要输出原文。
```

改为：
```
你是一个专业翻译助手。请直接将用户输入的文本从{sourceLang}翻译成{targetLang}，仅返回翻译结果，不要添加任何额外解释或说明，不要添加原文内容。
```

### 验证

- 翻译 "这里以6=2+2+2为例子" → 应输出 "Here we use 6=2+2+2 as an example"
- 翻译普通中文句子 → 行为不变
