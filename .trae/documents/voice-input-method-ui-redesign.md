# 语音输入法 UI 重设计方案

## 现状问题

| 问题   | 现状             | 目标         |
| ---- | -------------- | ---------- |
| 背景太暗 | `#09090b` 近乎纯黑 | 明亮、有呼吸感的背景 |
| 配色压抑 | 深灰 + 琥珀色，对比度低  | 活力配色，鲜明对比  |
| 卡片沉闷 | 暗色实心卡片         | 轻盈玻璃态/渐变卡片 |
| 缺乏动感 | 动画仅限录音状态       | 全局微交互、流畅过渡 |
| 整体氛围 | 被动、工具感         | 活跃、对话感     |

## 设计方向：Fresh Energy

**视觉论点**：温暖日光下的对话窗口 — 明亮、通透、有温度的语音交互体验

### 配色方案

```
背景：     #f8f6f3 (暖白) → #fef3e2 (暖杏渐变)
卡片：     rgba(255,255,255,0.7) 玻璃态 + 微妙阴影
主强调色：  #ff6b35 (活力橙)
次强调色：  #0ea5e9 (天际蓝)
成功色：   #22c55e (翠绿)
文字主色：  #1e1b18 (暖黑)
文字次色：  #6b6560 (暖灰)
文字弱色：  #a8a29e (浅暖灰)
边框：     rgba(0,0,0,0.06)
```

### 字体方案

* 标题：`Outfit` — 几何感、现代、有活力

* 正文：`Plus Jakarta Sans` — 温暖、易读、有个性

### 交互设计

1. **录音按钮**：渐变橙 → 录音时脉冲扩散 + 跳动波形
2. **卡片**：玻璃态 + 悬浮微升 + 阴影加深
3. **语言切换**：翻转动画
4. **翻译加载**：渐变色条流动
5. **页面加载**：元素依次淡入上浮

## 实施步骤

### Step 1: 创建 HTML 原型文件

在项目根目录创建 `ui-preview.html`，包含完整的独立 HTML/CSS/JS，无需构建工具即可预览。

**文件内容包含：**

1. **Header** — 品牌标识 + 状态指示
2. **录音区域** — 中央大录音按钮 + 脉冲动画
3. **控制栏** — 语言切换 + 翻译开关
4. **双栏内容** — 识别结果 + 翻译结果（玻璃态卡片）
5. **底部提示** — 操作引导

**技术实现：**

* 纯 HTML + CSS + 内联 JS

* Tailwind CDN（仅用于快速原型）

* Google Fonts CDN（Outfit + Plus Jakarta Sans）

* CSS 自定义属性管理配色

* CSS 动画实现所有交互效果

* 模拟数据展示各状态

### Step 2: 用户确认效果

用户在浏览器中打开 `ui-preview.html` 预览效果，确认设计方向后，再应用到实际 React 项目中。

### Step 3（待确认）: 应用到 React 项目

根据用户确认的设计，修改以下文件：

* `tailwind.config.js` — 更新配色和字体

* `src/index.css` — 更新全局样式

* `src/App.tsx` — 更新布局和类名

* 各组件文件 — 更新样式类名

## 验证标准

* [ ] HTML 原型可在浏览器中直接打开预览

* [ ] 整体感觉明亮、活跃、有温度

* [ ] 录音按钮交互流畅、反馈明确

* [ ] 卡片轻盈不沉闷

* [ ] 配色和谐、对比度足够

* [ ] 移动端响应式基本可用
