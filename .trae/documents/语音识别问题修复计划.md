# 语音识别问题修复计划

## 问题分析

### 问题 1：识别结果重复
**现象**：语音内容为"我正在使用翻译功能"，但识别出"我正在我正在使用我正在使用翻译功能我正在使用翻译功能"

**原因**：
- `continuous: true` 模式下，`onresult` 可能返回多个部分结果（interim 和 final）
- 当前代码在遍历 `event.results` 时未区分 `isFinal` 状态
- 每次回调都无条件拼接所有结果，导致重复

### 问题 2：Chrome 浏览器录制结束后不停止计时器
**现象**：结束录音后，内容没有识别出来，录制时间还在计时

**原因**：
- `recognition.onend` 事件在 Chrome 中可能没有正确触发
- 或者 Recognition 对象在停止时处于异常状态
- 计时器清理逻辑可能在 `onend` 之外没有其他保障

## 修复方案

### 修复 1：识别结果去重
在 `onresult` 回调中：
1. 检查每条结果是否是最终结果 (`isFinal`)
2. 只处理最终结果，或者在处理中间结果时去重
3. 记录已处理的 `resultIndex`，避免重复处理

### 修复 2：确保停止后正确清理
1. 在 `stopRecording` 函数中显式停止 Recognition 和清理计时器
2. 不依赖 `onend` 回调来清理资源
3. 添加状态保护，防止重复停止

## 具体代码修改

### 文件：`src/hooks/useVoiceRecognition.ts`

#### 修改 1：onresult 回调添加去重逻辑
```typescript
recognition.onresult = (event: any) => {
  let finalTranscript = '';
  for (let i = event.resultIndex; i < event.results.length; i++) {
    if (event.results[i].isFinal) {
      finalTranscript += event.results[i][0].transcript;
    }
  }
  if (finalTranscript) {
    setTranscript(prev => prev + finalTranscript);
  }
};
```

#### 修改 2：stopRecording 显式清理
```typescript
const stopRecording = useCallback(async () => {
  const recognition = recognitionRef.current;
  if (!recognition) return;

  setIsProcessing(true);
  
  if (timerRef.current !== null) {
    clearInterval(timerRef.current);
    timerRef.current = null;
  }
  
  recognition.stop();
};
```

#### 修改 3：添加状态保护
```typescript
const isStoppingRef = useRef(false);

const stopRecording = useCallback(async () => {
  if (isStoppingRef.current) return;
  isStoppingRef.current = true;
  
  // ... existing logic ...
  
  isStoppingRef.current = false;
}, []);
```
